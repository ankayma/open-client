# Ankayma — User Guide

How to use each part of the Ankayma app. Ankayma puts your machines and services on
a private mesh: traffic goes **directly between your devices** (encrypted, peer-to-peer)
and never through the vendor — the control plane only brokers identity and policy.

## Sections

| Guide | What it's for |
|---|---|
| [Devices](devices.md) | Enroll, view, and remove the machines on your mesh |
| [Services](services.md) | See which private services you're allowed to reach |
| [Subdomains](subdomains.md) | Give a service a private name (`name.you.int.ankayma.com`) |
| [SSH](ssh.md) | SSH into your machines with no bastion and no static key |
| [Members](members.md) | Invite teammates and manage roles (team tier) |
| [Access](access.md) | Author who-can-reach-what policy (team tier) |
| [Deploy Rules](deploy-rules.md) | Secretless CI/CD deploy from GitHub/GitLab |

## Tiers in one line

- **F0 (free):** your own devices, one of you, private subdomains, secretless deploy taste.
- **F1-Starter (team):** invite up to 10 members, roles, and access policy.

## A note on honesty

Where a feature isn't fully live yet, the guide says so plainly — Ankayma marks
unfinished edges rather than hiding them.
