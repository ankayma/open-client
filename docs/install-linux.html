Install · Linux & VPS

Run the mesh on any Linux box

One static binary. No glibc, OpenSSL, or system WireGuard to install — the agent ships its own userspace WireGuard. Works on Ubuntu, Debian, Alpine, RHEL, and any VPS, on every kernel from 4.x up.

x86_64 static musl Cosign-signed open source

Quick install

From a shell on the target machine (a server, VPS, or your dev box):

$ curl -fsSL https://get.ankayma.com/install.sh | sh

The script will:

  • detect your CPU architecture,
  • download the mesh CLI and the agent daemon,
  • verify their SHA-256 against a signed checksum manifest,
  • verify that manifest's Cosign signature (when cosign is installed), and
  • install both into /usr/local/bin.
Reading before piping to a shell is the right instinct — that's the point of an open client. Read it first: curl -fsSL https://get.ankayma.com/install.sh | less, or use the manual steps below.

Manual install & verification

If you'd rather not pipe a script, do every step by hand. This is also how you get the strongest supply-chain guarantee.

1. Download

# pick your version, e.g. latest $ base=https://get.ankayma.com/latest $ curl -fLO $base/mesh-linux-amd64 $ curl -fLO $base/agent-linux-amd64 $ curl -fLO $base/SHA256SUMS $ curl -fLO $base/SHA256SUMS.sig $ curl -fLO $base/cosign.pub

2. Verify integrity (checksums)

$ sha256sum -c SHA256SUMS mesh-linux-amd64: OK agent-linux-amd64: OK

3. Verify authenticity (Cosign signature)

One signature over SHA256SUMS authenticates every binary it lists — the same key and offline-verification model used for our GitLab releases.

$ cosign verify-blob --insecure-ignore-tlog \ --key cosign.pub --signature SHA256SUMS.sig SHA256SUMS Verified OK
No cosign? Install it from sigstore.dev. The checksums alone (over HTTPS) give you integrity; the Cosign step adds publisher authenticity. cosign.pub is committed in the open-client repo — compare it, don't just trust the copy you downloaded.

4. Install

$ sudo install -m 0755 mesh-linux-amd64 /usr/local/bin/mesh $ sudo install -m 0755 agent-linux-amd64 /usr/local/bin/agent

First run

Generate a WireGuard keypair (the CLI mirrors wg(8), so output is interchangeable with the stock tools):

$ mesh genkey | tee priv.key | mesh pubkey <your-node-public-key>

Bring the overlay up. On Linux the agent uses the kernel TUN device (/dev/net/tun), so it needs CAP_NET_ADMIN — run it with sudo (or as a service):

$ sudo agent up
Running unprivileged CI/CD? The agent also has a userspace-network deploy path (agent ci-deploy) that moves traffic over the tunnel without a kernel TUN device — so it runs inside an unprivileged hosted-CI container.

Run it as a service (server / VPS)

For an always-on node, run the daemon under systemd. Create /etc/systemd/system/ankayma-agent.service:

[Unit] Description=Ankayma mesh agent After=network-online.target Wants=network-online.target [Service] ExecStart=/usr/local/bin/agent up Restart=on-failure AmbientCapabilities=CAP_NET_ADMIN NoNewPrivileges=yes [Install] WantedBy=multi-user.target
$ sudo systemctl daemon-reload $ sudo systemctl enable --now ankayma-agent

Update & uninstall

Re-run the installer to update — it overwrites the binaries in place:

$ curl -fsSL https://get.ankayma.com/install.sh | sh

To remove:

$ sudo systemctl disable --now ankayma-agent 2>/dev/null || true $ sudo rm -f /usr/local/bin/mesh /usr/local/bin/agent $ sudo rm -f /etc/systemd/system/ankayma-agent.service

Build from source

The agent is open source — you don't have to trust our binaries. The toolchain is pinned (rust-toolchain.toml), so builds are reproducible:

$ git clone https://github.com/ankayma/open-client && cd open-client $ cargo build --release --target x86_64-unknown-linux-musl \ --bin mesh --bin agent

Current limits

We publish what isn't done yet rather than imply otherwise:

  • x86_64 only. arm64 (aarch64) builds aren't published yet.
  • CLI is intentionally small. mesh today covers key tooling (genkey, pubkey); node lifecycle lives in the GUI and the agent daemon. More CLI surface lands as the product matures.
  • See Honest limits for the full picture.
