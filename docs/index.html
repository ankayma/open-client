How it works

From install to tunnel in under 5 minutes

No configuration files, no manual key exchange, no firewall rules.

1
Sign up with GitHub One click. Your GitHub account gates access — no new password to manage.
2
Keys generated on your device WireGuard keypair created locally. Private key never leaves your device — not even to us.
3
Node joins your mesh Your device gets a stable overlay IP and a signed identity cert under your own tenant CA.

Add more devices Install on any other device, sign in with the same GitHub account. Encrypted P2P tunnel up automatically.
Features

Zero trust, not zero control

Designed for developers who want security without giving up auditability.

🔒
WireGuard end-to-end

Modern cryptography. Data travels peer-to-peer, encrypted, never through our servers.

🔑
Your keys, your CA

Each tenant gets its own Certificate Authority. We sign your node certs; we can't impersonate your nodes.

📖
Open-source agent

The code running on your machines is fully auditable. Reproducible builds, Cosign-signed artifacts.

🌐
Edge channel (*.mesh.dev)

Expose any service in your mesh via a stable subdomain. No public IP, no port forwarding needed.

⚡
NAT traversal built-in

STUN + hole punching + relay fallback. Works behind NAT, firewalls, and mobile networks.

🛡️
Default-deny policy

Every connection requires explicit policy approval. No implicit trust between nodes.
